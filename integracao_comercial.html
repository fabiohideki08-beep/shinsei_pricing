🎯 Objetivo de precificação padrão

Define o critério principal que o motor usa para calcular o preço ideal. Pode ser sobrescrito no simulador para casos específicos.

💵
Lucro líquido
Garante R$ X ou X% de lucro após todos os custos
📊
Margem
% do preço de venda que sobra como lucro
✖️
Markup
Multiplica o custo por um fator (ex: 2x = 100%)
Adicionar à fila automaticamente ao calcular
Sempre que o simulador calcular um preview válido, envia automaticamente para a fila
Embalagem padrão (R$)
Custo médio de embalagem por produto. Entra no custo base de todos os cálculos.
Usar taxas ML em tempo real
Busca comissão e frete operacional direto da API do Mercado Livre por produto.

💰 Gordura promocional por canal

Define o acréscimo aplicado sobre o preço calculado para criar o Preço Virtual (preço âncora).
O preço calculado vira o Preço Promocional no marketplace — criando a percepção de desconto.

🤖 Regras de automação por margem

Quando o sistema roda o scheduler, define o que fazer com base na margem calculada.

✅ Acima disso → aplica direto
⏳ Entre os dois → vai para fila
🚫 Abaixo da fila → bloqueia

Forças dos canais (peso no cálculo de score)

Influencia qual canal o sistema prioriza. Valores entre 0 e 1.

📦 Regra de estoque baixo

Quando o estoque de um produto estiver abaixo do limite, o sistema pode aumentar o preço automaticamente para proteger a margem.

Ativar regra de estoque
Quando ativo, produtos com estoque baixo terão preço ajustado
⚠️ Com estoque ≤ 2 unidades, o preço será aumentado em 0 para proteger o estoque restante.
